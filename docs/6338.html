获取您的k0s开发集群

原文：https://itnext.io/get-your-k0s-development-cluster-7491a1c76aca?source=collection_archive---------3-----------------------#2021-10-19

"如果你能运行Lens，你将拥有开发集群，永远可用！"

如果你经常使用Kubernetes，你可能会从 Mirantis 中看到Lens Kubernetes IDE。我个人使用Lens，并向任何利用Kubernetes的人推荐它，无论你是在操作方面还是在应用程序管理方面。

就在最近(几天前)，Lens团队宣布了Lens Spaces的一个新的alpha功能:用户现在可以直接从Lens运行和访问免费的开发集群。该开发集群是一个单节点集群，运行在AWS上的一个2 vCPUs、8gb RAM节点上(集群位置将来可能会改变)。由于该功能仍处于测试阶段，许多变化和增强将很快出现。官方声明可在这里获得。

在这篇快速的文章中，我们将看看启动Dev集群的所有必要步骤(提示:非常简单👍)

ℹ️由于该功能是alpha版，目前只能通过邀请访问。

一旦您获得访问权限，您将需要登录到您的Lens Spaces帐户，并选择新的个人空间环境。从那里，您可以进入群集目录，并立即开始调配开发群集。

只需几分钟，群集就可以启动并运行。与此同时，您将大致了解一下集群特性及其特征。

配置完成后，您可以立即连接到集群，全面了解集群中发生的一切。

现在让我们在集群中运行一个新的工作负载。我们将使用以下命令创建一个简单的 ghost映像和相关的服务:

$ kubectl run ghost --image=ghost:4 --expose --port 2368
service/ghost created
pod/ghost created

Dev集群可通过安全的端到端隧道和加密经由Lens Spaces进行访问，并且不向外界公开。为了访问上面创建的服务，您需要运行端口转发。

$ kubectl port-forward svc/ghost 8080:2368

由于Dev集群位于您的个人镜头空间中，您可以轻松地与任何地方的团队成员共享该集群。可以通过用户名/电子邮件地址或可共享的链接进行共享。通过共享空间共享集群是一个很棒的功能，可以用于演示、调试和许多其他事情。

另一方面(感谢 Edward Ionel 接受我的分享请求:)，新成员可以看到集群并在其中导航。
