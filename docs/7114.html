您是否曾经遇到过这样的情况:您需要通过SSH访问一组EC2实例，但是您没有访问权限，因为原始所有者已经离开，并且相关的私有密钥不可用(因为初始设置后私有密钥不可访问)？或者，您可能想使用像 Ansible 这样的工具，从一个专用的网络访问中对一组EC2实例运行自动化。也许您正在设置一个CI/CD管道，并且需要建立主节点和工作节点之间的通信。本文将使用基础设施作为代码，为这种设置介绍一种可重复的机制。

AWS CDK

第二版亚马逊网络服务云开发套件(CDK) 于去年(2021年)上市。CDK允许开发者使用一些最流行的编码语言来开发云形成模板；AWS的基础设施作为代码规范。编写云形成模板(CFT)可能是一项繁重的任务，但通过CDK，开发人员可以使用API来生成、部署和销毁CFT。本演练中显示的示例是用Python编写的，但可以转换为您选择的受支持语言。

循序渐进

让我们看一个基于CDK的项目的例子，它将:
1。创建虚拟专用网络(VPC)

2.生成用于访问EC2实例的密钥对，该EC2实例将通过 AWS Lambda函数创建

3.将私钥存储在 AWS秘密管理器中

4.创建一个可以使用生成的密钥对访问的自动缩放组

5.创建一个manager EC2实例来捕获系统上的私钥，以便可以使用它来访问自动缩放组中的实例。
